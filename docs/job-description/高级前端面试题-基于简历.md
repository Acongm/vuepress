## 一、系统重构与架构设计（必问）

### 1. 用户中心重构项目

**Q1: 你提到整合了 PC/移动端代码，维护成本降低 50%。具体是如何实现响应式融合开发的？**

- 考察点：响应式设计、代码复用、移动端适配方案
- 期待答案：
  - CSS 媒体查询 + flexible 布局
  - 组件级别的响应式封装
  - 一套路由配置多端适配
  - viewport 适配方案

**Q2: LCP 从 2.3s 优化到 1.4s，具体做了哪些优化？请详细说明性能优化四板斧的实施细节**

- 考察点：性能优化、Core Web Vitals、构建优化
- 期待答案：
  - 骨架屏预渲染的 SSR/SSG 方案
  - core-js 按需引入，Tree Shaking
  - 字体压缩（font-spider、woff2 格式）
  - TLS 1.3 的配置
  - 资源加载优先级（preload/prefetch）
  - Gzip/Brotli 压缩配置
  - WebP 图片兼容方案（picture 标签 + srcset）

**Q3: Vue3 + Vite + Pinia 的技术选型理由是什么？相比 Vue2 + Vuex 有哪些优势？**

- 考察点：技术选型能力、Vue3 特性、状态管理
- 期待答案：
  - Composition API 的优势
  - Vite 的快速冷启动
  - Pinia 更轻量、TypeScript 支持更好
  - Tree Shaking 效果更好

## 答题建议

### 回答结构（STAR 原则）

- **S**ituation：背景情况
- **T**ask：任务挑战
- **A**ction：采取行动
- **R**esult：结果成果

### 答题技巧

1. **数据量化**：尽量用数据说话（如：提升 40%、降低 50%）
2. **技术深度**：展示对技术原理的理解
3. **业务价值**：说明技术方案对业务的帮助
4. **反思总结**：分享经验教训和改进思路
5. **举一反三**：展示知识迁移能力

### 准备建议

1. 梳理每个项目的技术难点和解决方案
2. 准备性能优化的量化数据和对比图表
3. 整理代码示例和架构图
4. 准备技术分享的 PPT 或文章链接
5. 思考项目中的不足和改进空间
