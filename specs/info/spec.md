# Spec

**Goal**:  
基于我的前端开发简历，系统梳理所有涉及的技术栈与项目经验，提炼出结构化、原理导向的前端知识点清单，输出为 Markdown 文档，用于面试准备与技术复盘。

**Context**:  
我是一名前端工程师，主导过基于 React + Ant Design 的动态表单系统、Backstage 插件开发、微前端架构、组件库（UMD/ESM 双输出）、低代码表单引擎（无 Formily）等项目。需要将“做过什么”转化为“懂什么、为什么这么做”。

**Constraints**:

- 输出必须为纯 Markdown 格式，支持 VS Code 预览。
- 每个知识点需包含：定义、核心原理、项目上下文、典型面试问题。
- 聚焦“原理”而非 API 使用。
- 按技术领域分类，具备清晰导航结构。

**Output**:

- 文件名：`frontend-knowledge-map.md`
- 包含章节：工程化、React 原理、表单系统、微前端、TypeScript、构建工具、性能优化等。
