# 高级前端面试题 - 基于彭聪简历

根据简历内容整理的面试题，覆盖实际项目经验和技术栈

# 一、系统重构与架构设计（必问）

# 1. 用户中心重构项目

Q1: 你提到整合了 PC/移动端代码，维护成本降低 50%。具体是如何实现响应式融合开发的？

  • 考察点：响应式设计、代码复用、移动端适配方案
  • 期待答案：
    • CSS 媒体查询 + flexible 布局
    • 组件级别的响应式封装
    • 一套路由配置多端适配
    • viewport 适配方案

Q2: LCP 从 2.3s 优化到 1.4s，具体做了哪些优化？请详细说明性能优化四板斧的实施细节

  • 考察点：性能优化、Core Web Vitals、构建优化
  • 期待答案：
    • 骨架屏预渲染的 SSR/SSG 方案
    • core-js 按需引入，Tree Shaking
    • 字体压缩（font-spider、woff2 格式）
    • TLS 1.3 的配置
    • 资源加载优先级（preload/prefetch）
    • Gzip/Brotli 压缩配置
    • WebP 图片兼容方案（picture 标签 + srcset）

Q3: Vue3 + Vite + Pinia 的技术选型理由是什么？相比 Vue2 + Vuex 有哪些优势？

  • 考察点：技术选型能力、Vue3 特性、状态管理
  • 期待答案：
    • Composition API 的优势
    • Vite 的快速冷启动
    • Pinia 更轻量、TypeScript 支持更好
    • Tree Shaking 效果更好

# 2. RMS 系统重构项目

Q4: 100+页面的大型项目，如何保证多人协作的代码质量？

  • 考察点：团队协作、代码规范、工程化
  • 期待答案：
    • ESLint + Prettier 配置
    • Git commit 规范（Commitlint）
    • Code Review 流程
    • 组件库和设计规范
    • 单元测试覆盖率要求

Q5: 低代码平台落地，页面开发效率提升 40%。低代码平台的架构是怎样的？

  • 考察点：低代码设计、元数据驱动、动态渲染
  • 期待答案：
    • JSON Schema 配置化
    • 组件注册和动态渲染机制
    • 表单/表格配置化
    • 拖拽编辑器实现
    • 代码生成器

Q6: 日均处理 1W+工单，如何保证系统稳定性和性能？

  • 考察点：高并发处理、性能优化、监控告警
  • 期待答案：
    • 虚拟滚动（万级数据）
    • 请求防抖节流
    • 分页加载策略
    • 前端错误监控（Sentry）
    • 性能监控指标

# 3. 跨平台公告插件系统

Q7: 如何将包体积控制在 15KB 以内？Rollup Tree Shaking 的实践经验？

  • 考察点：打包优化、Tree Shaking、包体积控制
  • 期待答案：
    • Rollup 配置（ES Module 输出）
    • 副作用声明（sideEffects）
    • 按需加载
    • 移除开发代码和注释
    • 压缩和混淆

Q8: Preact 信号机制是什么？为什么选择 Preact 而不是 React？

  • 考察点：框架原理、性能优化
  • 期待答案：
    • Preact Signals 响应式原理
    • 相比 React 体积优势（3KB vs 40KB）
    • 性能对比
    • API 兼容性

Q9: 如何实现热插拔架构？接入周期<0.5 人日是如何做到的？

  • 考察点：插件化架构、模块化设计
  • 期待答案：
    • 插件注册机制
    • 生命周期钩子
    • 配置化接入
    • 标准化文档

Q10: CDN/Iframe/Web Component 三种植入方案的优劣对比？

  • 考察点：Web Components、微前端、隔离方案
  • 期待答案：
    • CDN：简单快速，但样式可能冲突
    • Iframe：完全隔离，但通信复杂
    • Web Component：标准化，浏览器支持情况

# 答题建议

# 回答结构（STAR 原则）

  • Situation：背景情况
  • Task：任务挑战
  • Action：采取行动
  • Result：结果成果

# 答题技巧

  1. 数据量化：尽量用数据说话（如：提升 40%、降低 50%）
  2. 技术深度：展示对技术原理的理解
  3. 业务价值：说明技术方案对业务的帮助
  4. 反思总结：分享经验教训和改进思路
  5. 举一反三：展示知识迁移能力

# 准备建议

  1. 梳理每个项目的技术难点和解决方案
  2. 准备性能优化的量化数据和对比图表
  3. 整理代码示例和架构图
  4. 准备技术分享的 PPT 或文章链接
  5. 思考项目中的不足和改进空间
